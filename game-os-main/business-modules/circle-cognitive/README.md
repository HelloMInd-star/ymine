## 🧬 理论溯源：运动-认知耦合

画圈实验起源于认知神经科学**运动-认知耦合理论**，依靠手部画圈动作激活小脑与运动皮层，搭建**预期-反馈时序闭环**。

### 三大基础变量全局统一映射

| 物理变量 | 业务映射 | 坐标轴 |
|----------|----------|--------|
| 🔵 圆圈半径 | **供给** | 水平轴 (X) |
| 🟣 画圈运行速度 | **需求** | 垂直轴 (Y) |
| 🟡 圆心偏移/方向约束 | **政策外部干预** | 全局偏移 |

系统每一次外部扰动，代表整套系统重新完成**均衡校准**。

### 核心架构公式：3→1→1→3→100

```
3 → 1 → 1 → 3 → 100
│   │   │   │   └── 100个全行业可拓展应用场景
```

## 🎮 三大运行模式

### 模式一：📐 数学模式（静态固定圈层）

- **特征**：圆心、半径完全固定，速度 = 0（属于动态圈的特殊形态）
- **核心载体**：鸡兔同笼二元线性方程组模型
- **变量映射**：
  - 鸡数量 = 需求
  - 兔数量 = 供给
  - 总头数 = 弹性约束
  - 总腿数 = 政策硬性约束
- **功能**：可视化两条直线相交、自动计算唯一均衡解、展示数值结果

### 模式二：🌊 经济模式（动态演化圈层）

- **特征**：圆心可漂移、半径可伸缩、全程持续旋转运行
- **波动机制**：需求沿垂直轴波动，供给沿水平轴波动，政策直接控制圆心整体偏移
- **黑天鹅扰动**：外部冲击滑块，产生视觉冲击波、瞬间偏移均衡圆心
- **两套市场完整模拟**：

| 市场类型 | 特征 |
|----------|------|
| 🌊 蓝海市场 | 低波动、高均衡稳定性、高额利润空间 |
| 🔥 红海市场 | 剧烈扰动、稳定性差、利润被持续压缩 |

- **输出指标**：均衡稳定性、价格波动幅度、整体利润空间、博弈收敛速度

### 模式三：🤖 AI 智能体反馈循环圈层

- **变量映射**：
  - 用户输入提示词 = 需求端
  - AI 生成内容 = 供给端
  - 上下文长度 = 系统弹性
  - 全局系统指令 = 政策约束
  - Token 生成速率 = 运行速度
- **完整链路**：输入 → 推理输出 → 反馈修正 → 新一轮迭代逼近均衡

## 🏗️ 工程架构：三层刚性对齐

本模块严格遵循 GameOS 三层刚性架构：

```
┌─────────────────────────────────────────────────────────┐
│  L1 表层：可视化交互页面                                  │
│  位置：game-os-main/ (公开主仓库)                         │

│  职责：UI渲染、用户交互、结果展示、空调用逻辑              │
│  ⚠️  所有私有参数置空 null                                │
└──────────────────────┬──────────────────────────────────┘
                       │ 标准API通道 (circle_前缀)
┌──────────────────────▼──────────────────────────────────┐
│  L2 中间层：统一 GameOS 公共API调度层                     │
│  位置：business-modules/circle-cognitive/private-logic/  │
│  文件：circle-public-api.js                              │
│  职责：参数校验、熔断检查、统一响应格式、事件总线          │
└──────────────────────┬──────────────────────────────────┘
                       │ 私有引擎载入（上锁）
┌──────────────────────▼──────────────────────────────────┐
│  L3 算力层：私有认知引擎仓库                              │
│  位置：ymine-circle-cognitive-engine/ (私有GitHub仓库)   │
│  职责：加权算法、拟合系数、蓝海/红海阈值、偏差权重         │
│  🔒 上锁保管，不对外暴露                                  │
└─────────────────────────────────────────────────────────┘
```

### 公私分离原则

| 公开侧 (game-os-main) | 私有侧 (ymine-circle-cognitive-engine) |
|----------------------|---------------------------------------|
| 界面、交互、空调用逻辑 | 全部加权算法、拟合系数 |
| 所有参数置空 `null` | 蓝海/红海阈值、偏差权重 |
| 仅通过标准API获取结果 | 创始人五大维度权重、PDCA系数 |
| 前端**严禁**直接读写私有引擎源码 | 认知循环映射系数、认知路径系数 |

## 🛡️ 熔断机制

全局兼容 **0.6 风险浓度熔断机制**：

- 触发阈值：`riskConcentration > 0.6`（黑天鹅扰动 ≥ 60/100）
- 触发行为：操作自动锁死，返回 HTTP 403
- 状态上报：通过 `GameOSBus.emit('circle-fuse-update')` 同步总控台全局状态
- 双重防护：前端滑块交互检测 + API层校验

**配色规范**：
- 🟢 绿色 = 安全
- 🟡 黄色 = 预警
- 🔴 红色 = 熔断告警

## 🚀 使用教程

### 快速开始

1. **进入模块**：从Y.MINE总控台左侧侧边栏「🧠 画圈实验」入口，或点击核心方法论系统中的「画圈实验 · 脑神经到商业决策」卡片进入
2. **选择模式**：顶部四个Tab切换不同运行模式
   - 📐 数学模式：调整总头数/总腿数滑块，点击「求解均衡」
   - 🌊 经济模式：选择蓝海/红海，调整供需/政策/黑天鹅滑块，点击「启动演化」
   - 🤖 AI反馈模式：输入提示词，设置上下文长度，启动反馈循环
   - 🎯 实验对照：点击「开始画圈」后在Canvas上画圆进行轨迹采集
3. **查看结果**：Canvas可视化 + 右侧指标卡片 + 下方三模块（偏差对照、匹配度、市场预判）

### 实验流程建议

1. 先在「📐 数学模式」理解静态均衡的确定性求解
2. 切换到「🌊 经济模式」观察蓝海/红海的动态演化差异，体验黑天鹅扰动
3. 「🤖 AI反馈模式」体验认知循环的迭代收敛
4. 「🎯 实验对照」亲身画圈，采集自己的神经运动指标
5. 使用下方「⚖️ 双决策对照组」对比理性vs人脑偏差
6. 调节「👤 创始人匹配度」五维滑块，评估自己与商业模型的匹配
7. 在「🔮 前置市场预判」选择你的预判风格

## 🔐 私有引擎依赖说明

### 保密红线（硬性要求）

1. ❌ 核心算法、私有权重**禁止对外暴露**
2. ❌ 所有空白参数必须标注：**【需在私有circle认知引擎内配置填充】**
3. ❌ 不得改动 GameOS 底层已经锁死的 Core 公理规则
4. ❌ 前端严禁硬编码任何私有数值，全部依赖私有引擎外部载入
5. ❌ 所有运算只能走统一 API 通道，禁止绕过中间层

### 私有引擎需填充的权重项

| 权重分类 | 参数项 |
|----------|--------|
| 创始人匹配权重 | founderAbility, decisionQuality, cognitiveAccuracy, marketFit, executionEfficiency |
| 认知偏差权重 | anchoring (锚定), lossAversion (损失厌恶), overconfidence (过度自信) |
| 市场阈值 | blueSeaVolatility, redSeaVolatility, convergenceSpeed, profitCompression, blackSwanRecoveryRate |
| 神经运动阈值 | roundnessPerfect, roundnessGood, radialErrorMax, speedVariationMax, driftTolerance |
| PDCA系数 | planTargetTolerance, doExecutionVariance, checkFeedbackGain, actLearningRate |
| 认知循环 | learningDecayRate, contextResetThreshold, retrievalAccuracy, generationQuality, iterationConvergence |
| 认知路径 | linguisticMapping, mathProblemSolving, formalDeduction |

私有引擎载入后需设置：
```javascript
window.PRIVATE_ENGINE.connected = true;
```
公共API层会自动检测并使用私有权重覆盖占位默认值。

## 🔗 与十步闭环流水线对接
