## 二、系统概述

### 2.1 系统定位

MindSpeak V19.0 是一套面向自然语言结构化解析的四层跨域数理同构翻译引擎，核心价值在于建立"人类自然语言"与"数理算法结构"之间的严谨数学映射，使自然语言输入可被计算机系统精确理解并进入量化计算流水线。

本引擎作为 GameOS 十步流水线 **S1（ETL信息清洗环节）** 的标准自然语言结构化解析入口，所有进入 GameOS 主系统的自然语言输入必须经由 MindSpeak 完成向量化和结构化转换。

### 2.2 设计目标

1. **跨域同构：** 在语言层、数理层、算法层之间建立严格的数学同构映射关系
2. **可视化：** 所有映射过程通过Canvas实时动态演算可视化，三层结构肉眼可辨
3. **可审计：** 每一步推导都有完整日志记录，所有结果强制三角三模型并行校验
4. **公私隔离：** 公开演示层与涉密精算内核严格物理隔离，满足软著交存要求
5. **阈值统一：** 严格遵循GameOS全域归一化规范，0.48/0.5/0.68三级阈值

### 2.3 核心约束（强制锁死）

- **数值区间：** 全域归一至 [0, 1] 闭区间
- **保本安全线：** 0.48
- **稳态中轴线：** 0.50
- **熔断红线：** 0.68
- **审计容差：** ±0.02
- **全局版本：** Game-OS V2.1，MindSpeak V19.0
- **统一逻辑范式：** 生成候选 → 验证约束 → 冲突回退 → 采纳最优

## 三、总体架构设计

### 3.1 三层映射架构

本软件采用三层联动架构，用户输入后三层同步刷新：

```
┌─────────────────────────────────────────────────────────────┐
│  【语言表达层】Layer 1                                        │
│  原始文本输入、对照样本载入、模块参数配置、输出文本展示       │
└──────────────────────┬──────────────────────────────────────┘
                       │ 四维语义向量映射
                       ▼
┌─────────────────────────────────────────────────────────────┐
│  【数理结构层】Layer 2                                        │
│  Canvas动态可视化、置信度实时数值、阈值指示器、二级辅助图     │
└──────────────────────┬──────────────────────────────────────┘
                       │ 算法架构映射
                       ▼
┌─────────────────────────────────────────────────────────────┐
│  【算法架构层】Layer 3                                        │
│  演算流程图、模块细节说明、三角审计结果、三模型状态指示灯     │
└─────────────────────────────────────────────────────────────┘
```

### 3.2 系统模块划分

系统按功能划分为10个独立映射模块，每个模块对应一种语言学现象与一类算法/数学结构的同构映射：

| 编号 | 模块名称 | 语言现象 | 同构数理/算法结构 |
|------|---------|---------|------------------|
| ① | 语序映射 | 不同语言定语/修饰语语序差异 | 线性代数向量交换律 |
| ② | 时态映射 | 语法时态与持续时间 | 凯利概率公式模型 |
| ③ | 词缀映射 | 派生词缀（前缀/后缀/中缀） | 多层特征工程/特征拓扑 |
| ④ | 语态映射 | 主动句/被动句转换 | IoC控制反转/依赖注入 |
| ⑤ | 多义词映射 | 一词多义语境消歧 | 策略模式路由分支 |
| ⑥ | 文化映射 | 文化意象与禁忌 | L2正则化约束边界 |
| ⑦ | OOD检测 | 生词/伪同源/异常输入 | 分布外异常检测/置信度衰减 |
| ⑧ | 鸡兔同笼 | 经典算术应用题 | 二元一次方程组求解 |
| ⑨ | PACD工程 | 多人合作工程问题 | 并行任务调度速率叠加 |
| ⑩ | 枚举验证 | 组合枚举问题 | CSP约束满足剪枝算法 |

### 3.3 全局控制组件

系统包含以下全局控制组件，对所有模块生效：

- **面包屑导航：** 总控台→认知科研实验组→MindSpeak
- **全局稳态滑块：** ±0.2范围手动扰动参数，观察收敛/熔断
- **紧急停机/恢复按钮：** 一键切断/恢复所有运算
- **S0-S9流水线指示器：** 高亮S1当前位置
- **三模型状态指示灯：** CALC/GM/CMP实时校验状态
- **总览全景矩阵图：** 10模块×3层架构总览热力图
- **全流程演算日志：** 逐行打印，不省略中间步骤

## 五、十大功能模块详细设计

### 5.1 模块①：语序 ↔ 线性代数交换律

#### 5.1.1 模块说明
不同自然语言的定语/修饰语位置存在系统性差异（如英文形容词前置、马来语形容词后置、德语从句动词后置），这种语序差异在线性代数框架下等价于向量空间中基向量的位置交换，线性代数交换律在此线性空间内严格成立。

#### 5.1.2 输入输出
- **输入：** 任意自然语言句子
- **控制参数：** 是否执行语序交换（复选框）
- **输出：** 交换后的对应表达
- **数理映射：** 句法维度向量位置交换，交换律验证

#### 5.1.3 可视化方案
- 主Canvas：Adj/N两个节点沿弧线平滑交换位置的60fps动画
- 展示线性空间连通路径虚线

#### 5.1.4 对照样本（5组）
1. 中英定语前后置换（英文Adj顺序→中文定语前置）
2. 马来语定语后置（南岛语系核心名词前置反转）
3. 德语从句尾语序（动词后置位置向量映射）
4. 日语定语连体形（与中文同构，交换律验证）
5. 多重定语嵌套边界（分层交换律边界条件）

---

### 5.2 模块②：时态 ↔ 凯利概率模型

#### 5.2.1 模块说明
语法时态对应凯利（Kelly）公式中的胜率参数p和持仓周期（有效年龄）：
- 一般现在时 ↔ p≈0.55，稳态持仓
- 现在进行时 ↔ p≈0.62，动态调整
- 现在完成时 ↔ p≈0.75，累积胜率（接近熔断）
- 一般将来时 ↔ p≈0.45，保本线下观望

凯利公式：f* = 2p - 1（简化二元形式），其中f*为最优持仓比例。

#### 5.2.2 输入输出
- **输入：** 时态选择（下拉）、胜率p（滑块0.30~0.75）
- **输出：** 凯利最优仓位f*数值、状态说明
- **数理映射：** 时态维度→胜率参数，持续时长→持仓周期

#### 5.2.3 可视化方案
- 主Canvas：概率收益动态曲线，叠加0.48保本绿线、0.50稳态黄线、0.68熔断红线
- 凯利最优点实时标记并跟随p值移动
- 历史置信度轨迹曲线

#### 5.2.4 对照样本（5组）
1. 一般现在时（p=0.55，稳态持仓）
2. 现在进行时（p=0.62，动态调整区间）
3. 现在完成时（p=0.75，累积胜率，接近熔断）
4. 一般将来时（p=0.45，低于保本线，观望）
5. 边界p=0.70（触碰0.68红线附近，触发预警审计）

### 5.3 模块③：词缀派生 ↔ 特征工程

#### 5.3.1 模块说明
自然语言的词缀派生过程（词根加前缀/后缀产生新词）等价于机器学习中的多层特征变换过程：
- 词根 = 原始输入特征
- 词缀 = 特征变换算子
- 派生词 = 高阶交互特征
- 多层派生 = 多层神经网络特征拓扑

#### 5.3.2 输入输出
- **输入：** 印尼语词根（如ajar/tulis/baik）
- **输出：** 词缀派生链、语义特征列表
- **数理映射：** 多层特征变换拓扑网络

#### 5.3.3 可视化方案
- 主Canvas：三层特征拓扑网络图（基础层→变换层→输出层），节点发光，连线动态流动
- 节点颜色区分层级：青色→紫色→粉色

#### 5.3.4 对照样本（5组）
1. 印尼语名词派生（pe-ajar, pe-ajar-an, peng-ajar）
2. 印尼语动词派生（me-nulis, me-nulis-kan, di-tulis）
3. 印尼语形容词派生（ter-baik, se-baik-nya, ke-baik-an）
4. 简单基础特征（单层线性变换）
5. 高阶交互特征（多层非线性拓扑）

### 5.4 模块④：主动/被动 ↔ IoC依赖反转

#### 5.4.1 模块说明
主动句向被动句的语态转换等价于软件架构中的控制反转（Inversion of Control）模式：
- 原主语 → 从控制权拥有者降格为旁格（相当于被注入的依赖）
- 原宾语 → 提升为新主语（相当于控制权反转后的调用方）
- 被动标记"被/by" → 相当于依赖注入容器

#### 5.4.2 输入输出
- **输入：** 主动句或被动句文本
- **控制参数：** 是否执行IoC反转（复选框）
- **输出：** 反转后的对应语态句子
- **数理映射：** 数据流图方向反转，控制流依赖方向翻转

#### 5.4.3 可视化方案
- 主Canvas：三节点（主语-动词-宾语）之间箭头流动方向动态反转动画
- 绿色箭头=主动正向流，红色虚线箭头=被动反向流

#### 5.4.4 对照样本（5组）
1. 日常简单主动→被动（我吃苹果→苹果被我吃）
2. 日常被动→主动（双向反转验证）
3. 工程业务句式（服务器处理请求→请求由服务器处理）
4. 双宾语结构（老师教我们英语→两种被动路径）
5. 不及物动词边界（他来了→不可转换，边界条件触发）

### 5.5 模块⑤：多义词 ↔ 策略模式路由

#### 5.5.1 模块说明
多义词根据上下文语境选择不同释义的过程等价于软件设计模式中的策略模式（Strategy Pattern）：
- 多义词本身 = 策略上下文（Context）
- 上下文语境 = 路由键（Key）
- 每个释义 = 一个具体策略分支（ConcreteStrategy）
- 根据语境选择对应分支，禁用其他分支

#### 5.5.2 输入输出
- **输入：** 多义词文本、上下文
- **控制参数：** 策略分支选择（下拉选择）
- **输出：** 选定释义、路由分支编号
- **数理映射：** 语义维度one-hot向量路由到对应分支策略

#### 5.5.3 可视化方案
- 主Canvas：左侧入口节点发散出多条贝塞尔曲线连接右侧多个策略分支节点
- 当前激活分支高亮绿色，其他分支半透明

#### 5.5.4 对照样本（5组）
1. "打"+球类 → 体育分支（play）
2. "打"+交通工具 → 交通分支（hail/take）
3. "打"+液体 → 日常分支（fetch/draw）
4. "打"+抽象概念 → 法律分支（engage in）
5. 英文"bank"多义（河岸/银行/指望）三分支

### 5.6 模块⑥：文化正则化约束

#### 5.6.1 模块说明
不同文化背景下的意象差异和语言禁忌等价于机器学习中的L2正则化约束：
- 文化规范 = 正则化项，对转换结果施加边界惩罚
- 禁忌词 = 强约束，触发替换或加注机制

#### 5.6.2 输入输出

- **控制参数：** 文化约束强度（滑块0~1）
- **输出：** 约束说明、禁忌判定结果
- **数理映射：** 语义向量在文化约束方向上施加投影修正

#### 5.6.3 可视化方案

- 圆环颜色金黄，模拟文化约束场

#### 5.6.4 对照样本（5组）
1. 中国龙（正面祥瑞约束，禁止译邪恶义）
2. 西方Dragon（负面邪恶约束，禁止译祥瑞义）
3. 中国凤凰（雌雄同体，阴阳配对约束）
4. 西方Phoenix（单性涅槃，重生约束）

### 5.7 模块⑦：OOD边界异常检测

#### 5.7.1 模块说明
生词、伪同源词、网络新词、无意义字符串等价于分布外（Out-of-Distribution, OOD）样本：
- 训练集内词汇 → OOD距离小 → 置信度高 → 正常通过
- 伪同源/生僻字 → OOD距离中等 → 置信度下降 → 预警区
- 完全陌生词/乱码 → OOD距离大 → 置信度骤降 → 触发熔断

置信度衰减公式：conf = max(0, 1 - ood_score × 0.9)

#### 5.7.2 输入输出
- **输入：** 待测词汇
- **控制参数：** OOD距离（滑块0~1，模拟不同类型词汇）
- **输出：** 置信度数值、是否熔断
- **数理映射：** 输入向量与训练分布的距离度量→置信度衰减

#### 5.7.3 可视化方案
- 主Canvas：置信度衰减曲线，叠加三条阈值线
- 超阈值点红色圆圈闪烁告警
- 五个采样点动态绘制折线

#### 5.7.4 对照样本（5组）
1. 同源正常词汇（"美丽"，ood=0.15，conf=0.82，正常）

### 5.8 模块⑧：鸡兔同笼 ↔ 二元一次方程组

#### 5.8.1 模块说明
经典鸡兔同笼问题（头h个，脚f只，求鸡兔各多少）是二元一次方程组的典型应用：
- 方程组：
  - x + y = h（x=鸡数，y=兔数，头数约束）
  - 2x + 4y = f（脚数约束）
- 解法：代入消元 → y = (f - 2h) / 2，x = h - y
- 校验：x,y必须为非负整数，否则无解（异常检测触发）

#### 5.8.2 输入输出
- **输入：** 头数量（数字输入框）、脚数量（数字输入框）
- **输出：** 鸡数量、兔数量、是否有解
- **数理映射：** 算术应用题→线性方程组求解→整数约束验证

#### 5.8.3 可视化方案
- 主Canvas：鸡/兔数量动态柱状图，柱子高度随数值变化
- 无解时柱子灰化并显示异常标记

#### 5.8.4 对照样本（5组）
1. 经典题：35头94脚（23鸡12兔，《孙子算经》原题）
2. 小规模：10头28脚（6鸡4兔）
3. 大规模：100头280脚（60鸡40兔）
4. 边界值：1头4脚（0鸡1兔，全兔边界）
5. 异常值：35头93脚（奇数脚，无解，异常检测触发）

### 5.9 模块⑨：PACD工程效率 ↔ 并行调度

#### 5.9.1 模块说明
多人合作完成工程的问题（A单独a天完成，B单独b天完成，合作几天完成）等价于并行任务调度模型：
- 单人效率 rate = 1/天数
- 并行效率 = Σ rate_i（速率线性叠加）
- 合作工期 = 1 / 并行效率
- 阿马迪亚定律验证：最慢节点决定最大加速比

#### 5.9.2 输入输出
- **输入：** 人员配置选择（双人/三人）
- **输出：** 各单人效率、综合效率、合作工期、人员占比
- **数理映射：** 工程问题→并行速率叠加→瓶颈检测

#### 5.9.3 可视化方案
- 主Canvas：各人员效率占比环形饼图，中心显示综合工期
- 不同人员用不同颜色区分

#### 5.9.4 对照样本（5组）
1. 双人合作：A=10天，B=15天 → 合效率1/6 → 6天完成
2. 三人合作：A=10，B=12，C=15 → 合效率1/4 → 4天完成
3. 四人并行：边际收益递减验证
4. 先后接力vs完全并行对比
5. 瓶颈节点验证：最慢节点决定整体效率

### 5.10 模块⑩：枚举验证 ↔ 组合剪枝

#### 5.10.1 模块说明
组合枚举问题的验证过程等价于约束满足问题（CSP）中的剪枝算法：
- 纯暴力枚举：n! 复杂度（爆炸增长）
- 约束剪枝：通过约束传播大幅削减无效分支
- 剪枝率 = (总分支数 - 有效分支数) / 总分支数

数独、旅行商问题（TSP）等经典NP问题均依赖高效剪枝。

#### 5.10.2 输入输出
- **输入：** 枚举规模n（数字输入框）、是否启用剪枝（复选框）
- **输出：** 总分支数、剪枝后分支数、剪枝率
- **数理映射：** 枚举验证→CSP约束传播→分支定界剪枝

#### 5.10.3 可视化方案
- 主Canvas：枚举前/剪枝后分支数对数刻度柱状对比图
- 紫色柱=原始枚举，绿色柱=剪枝后

#### 5.10.4 对照样本（5组）
1. 3元素全排列：3!=6，无剪枝必要
2. 5选2组合：C(5,2)=10
3. 4元素带约束剪枝：24→10，剪枝率58%
4. 数独9×9：6.6×10^21→可解，约束传播
5. 旅行商（TSP）：10!→约1000，剪枝率>99.9%

## 六、全局联动机制设计

### 6.1 十步流水线接入

MindSpeak 严格接入 GameOS S0-S9 十步流水线：

```
S0 宏观海选 → S1 ETL信息清洗（MindSpeak）→ S2 核心定价估值 → S3 沙盘决策推演
→ S4 动态周期观测 → S5 人性偏差校验 → S6 人工调参确认 → S7 审计复核
→ S8 熔断风控对冲 → S9 返回总控台闭环
```

页面顶部流水线指示器中，S1 节点高亮为激活状态（青色），清晰标识 MindSpeak 在全局流水线中的位置。

### 6.2 三角三模型并行审计

所有运算结果在输出前**强制**送入三套独立模型并行校验，三套模型完全独立（代码隔离、存储隔离、运行实例隔离）：

| 模型代号 | 模型名称 | 模型说明 |
|---------|---------|---------|
| CALC | 金融精算模拟器 | 基于金融精算理论独立计算预期置信度 |
| GameMind (GM) | GameMind博弈沙盘 | 基于博弈论独立计算预期置信度 |
| Compute (CMP) | 几何心智算力底座 | 基于几何算力模型独立计算预期置信度 |

**审计判定三分支规则：**

1. **三模型偏差均 ≤ ±0.02：** 直接放行，结果采纳
2. **单组偏差 > ±0.02：** 触发均值修正，以三模型均值为最终值，写入预警日志
3. **两组及以上偏差 > ±0.02：** 运算阻断，显示红色审计失败提示，结果不予采纳

审计结果在算法架构层彩色区块实时显示，三模型状态指示灯（CALC/GM/CMP）实时显示✓/✗状态。

### 6.3 全局稳态扰动与收敛

全局稳态滑块允许用户在 ±0.2 范围内手动给系统施加扰动，观察系统行为：
- 扰动后置信度偏离0.5中轴线
- 三模型投票机制自动修正，系统向0.5稳态收敛
- 扰动过大导致置信度 > 0.68 时直接触发熔断锁定

滑块实时显示当前扰动值（带正负号），下方状态芯片实时更新（绿色稳态/黄色预警/红色熔断）。

### 6.4 紧急停机机制

- **紧急停机按钮：** 点击后立即设置 `emergency_stop=true` 标志位，持久化到 localStorage，所有后续运算调用直接拒绝，日志输出红色停止消息。
- **恢复运算按钮：** 点击后清除停机标志，恢复正常运算。
- 停机时顶部状态指示器变为红色"● 已停机"。

- `dim[3]` (confidence)：置信度维度——综合置信度评分

## 十、关键算法说明

### 10.1 统一逻辑范式

所有模块严格遵循统一四步处理范式：

```
1. 生成候选 → 基于输入生成若干可能的映射候选
2. 验证约束 → 应用语法/语义/文化/数学约束验证候选合法性
3. 冲突回退 → 不满足约束的候选回退至次优候选或报错
4. 采纳最优 → 选择置信度最高且通过所有约束的候选作为输出
```

### 10.2 鸡兔同笼求解算法

```
输入：heads（头数）, feet（脚数）
步骤：
1. 建立方程组：
   chicken + rabbit = heads
   2*chicken + 4*rabbit = feet
2. 代入消元：
   rabbit = (feet - 2*heads) / 2
   chicken = heads - rabbit
3. 合法性校验：
   - chicken ≥ 0 且 rabbit ≥ 0
   - chicken 和 rabbit 均为整数
   若不满足则判定为无解，触发异常检测
输出：{chicken, rabbit} 或 {null, null}（无解）
```

### 10.3 PACD效率叠加算法

```
输入：rates[] = [1/a, 1/b, 1/c, ...]（各单人效率，单位：项/天）
步骤：
1. 并行效率线性叠加：combined = Σ rates[i]
2. 合作工期：days = 1 / combined
3. 人员占比：distribution[i] = rates[i] / combined
输出：{combinedRate: combined, days: days, distribution: [...]}
```

### 10.4 置信度衰减算法（OOD）

```
输入：oodScore ∈ [0,1]（分布外距离）
公式：confidence = max(0, 1 - oodScore × 0.9)
状态判定：
  confidence > 0.68 → FUSED（熔断，因Y轴显示反转，0.68以上熔断）
  confidence > 0.48 → WARNING（预警区）
  否则 → STEADY（正常区，注意：此处阈值因显示Y轴反转做对应映射）
```

> 注：显示时Y轴从上到下为1→0，故置信度数值映射到Y坐标时使用 `y = h × (1 - confidence)`，0.68阈值线视觉位置在画面高度32%处。

### 10.5 三角审计均值修正算法

```
输入：三模型独立计算结果 {calc, gamemind, compute}
maxDiff = max(|calc-gm|, |gm-cmp|, |cmp-calc|)
若 maxDiff ≤ 0.02：result = PASS，采用原始置信度
若仅单模型偏差 > 0.02：result = CORRECTED，finalConf = (calc+gm+cmp)/3
若≥2个模型偏差 > 0.02：result = BLOCK，运算阻断
```

## 十一、异常处理与安全机制

### 11.1 数值钳位保护

所有置信度数值在输出前强制钳位至 [0, 1] 区间，防止越界：

```javascript
conf = Math.max(0, Math.min(1, conf));
```

### 11.2 紧急停机全局拦截

`MindSpeak.runMapping()` 入口第一时间检查 `isEmergencyStopped` 标志位，若已停机直接返回 `{error: 'EMERGENCY_STOPPED'}`，不执行任何后续运算。

### 11.3 审计日志不可篡改

- `audit_log_mindspeak` 分区仅提供追加接口，前端UI无删除/修改按钮
- 每条日志生成时附带唯一ID和时间戳
- 私有内核提供签名接口（signResult）可对关键结果签名防篡改

### 11.4 无框架纯原生实现

本软件全部采用原生HTML5+CSS3+ES6 JavaScript实现，无任何第三方框架依赖，避免供应链安全风险。所有代码本地静态托管即可运行，无需后端服务。

### 11.5 错误兜底

- Canvas初始化失败时静默降级，不阻塞UI交互
- localStorage不可用时自动降级为内存存储（日志不持久化但功能正常）
- 所有模块try-catch故障隔离，单模块异常不传染其他模块
