## 一、公私架构划分总则

### 1.1 划分原则

MindSpeak V19.0 严格遵循"公私有别、物理隔离、接口通信、源码不曝"十六字原则：

1. **目录物理二分：** 公开代码与涉密内核放置于完全独立的两个顶级目录，无交叉引用
2. **接口桩通信：** 公开层仅通过预先定义的密封接口桩调用私有内核功能，不直接访问内核源码
3. **Object.freeze 密封：** 所有通过window全局暴露的内核对象均使用Object.freeze()冻结，防止运行时篡改
4. **涉密内容零外泄：** 公开目录任何文件不得包含涉密权重数值、精算公式完整实现、加密密钥等内容

## 四、公私接口边界说明

### 4.1 调用方向约束

```
公开层 (mindspeak/)
    ↓ 仅通过window全局密封接口调用
    ↓ MindSpeakVector4D / MindSpeakWeights /
    ↓ MindSpeakConfidence / MindSpeakCrypto
私有内核层 (mindspeak-private-engine/)
```

**禁止反向调用：** 私有内核层不得依赖公开层任何变量或函数，保持单向依赖。

### 4.2 数据传递规则

1. **公开层→内核：** 传递原始输入向量（四维数组）、模块ID、参数对象等纯数据
2. **内核→公开层：** 返回处理后的结果对象（含dim数组、confidence数值、状态标记）
3. **禁止传递：** 不得将DOM元素、localStorage引用、函数对象等非纯数据传入内核
4. **禁止暴露：** 内核返回对象中不得包含任何函数引用或内部状态对象

### 4.3 Object.freeze 密封清单

所有暴露到window的内核对象均执行Object.freeze，防止运行时prototype污染或属性替换：

```javascript
// 每个内核文件末尾均执行
global.MindSpeakXxx = Object.freeze(XxxModule);
```

已密封对象清单：
- ✅ window.MindSpeakVector4D
- ✅ window.MindSpeakWeights

- ✅ window.MindSpeakCrypto
- ✅ window.MindSpeak（公开层引擎也一并冻结）

### 6.4 公开仓库中接口桩处理方案

GitHub公开仓库可以放入接口桩版本的mindspeak-private-engine/文件（即当前开发的桩版本），但：
1. 所有文件头部必须保留`_sealed: true`和`SEALED_FOR_COPYRIGHT_DEPOSIT`标记
2. 文件注释中明确说明"本文件为接口桩，完整精算逻辑属于涉密资产，已密封交存"
3. 桩文件中禁止出现任何涉密数值或算法完整实现
