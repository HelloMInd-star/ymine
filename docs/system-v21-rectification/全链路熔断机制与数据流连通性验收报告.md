## 1. 验收概述

### 1.1 验收范围

本次验收覆盖Game-OS V2.1全系统的熔断机制与数据流连通性，包括：

1. 全域统一阈值常量（0.48/0.50/0.68）在所有模块的存在性与一致性
2. 风控熔断七项通路的完整链路可用性
3. 从#30力学实验→PID控制→GameMind博弈→AirMind调度的长距离数据流连通性
4. 三分区存储隔离（pipeline/draft/audit_log）验证
5. 数据新鲜度30秒stale检查机制
6. 紧急停机按钮功能验证

### 1.2 验收标准

| 验收维度 | 通过标准 |
|---------|---------|
| 阈值一致性 | 所有模块中0.48/0.50/0.68三个阈值定义一致，无偏差 |
| 七项通路 | 每条通路检查项100%通过 |
| 数据流连通 | 从#30到AirMind的完整链路数据可正常流转 |
| 熔断执行 | C≥0.68时所有控件锁定，运算actualRatio强制为0 |
| 审计日志 | 每次熔断事件均写入audit_log |
| 数据新鲜度 | 超过30秒无更新显示stale警告 |
| 紧急停机 | 按钮点击后立即阻断所有运算 |

### 1.3 验收方法

- **静态代码检查**：全文搜索阈值常量定义位置与数值
- **动态功能测试**：在浏览器控制台注入测试用例验证触发逻辑
- **数据流追踪**：在各节点设置断点验证数据传递
- **localStorage检查**：验证各节点输出key存在性与数据格式
- **UI交互测试**：点击按钮、观察告警动效、验证控件禁用
