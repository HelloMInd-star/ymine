## 1. 文档概述与背景

### 1.1 文档目的

本说明书旨在系统性阐述Game-OS V2.1版本中全站导航体系与子引擎架构的整体规划方案。通过对导航层级、视觉呈现、交互逻辑、引擎定位等维度的标准化定义，确保全站架构的一致性、可扩展性与用户体验的连贯性。本文档可作为前端开发、UI设计、产品验收的权威依据，同时为项目验收与简历佐证提供正式架构说明。

### 1.2 编制背景

随着Game-OS系统从V2.0向V2.1迭代演进，系统内子引擎数量从3个扩展至6个（含几何心智空间算力底座），实验室模块从2类扩展至4类，原有的扁平式导航结构已无法承载当前系统复杂度。主要存在以下问题：

1. **导航层级混乱**：侧边栏在所有页面重复出现，造成视觉冗余与认知负担
2. **命名不统一**：同一模块在不同页面存在title、h1、面包屑、卡片名称不一致问题
3. **视觉差异化不足**：各子引擎缺乏专属视觉识别系统，用户难以快速区分
4. **层级关系模糊**：GameMind作为公共基座与四大平行子引擎的视觉层级未做区分
5. **返回路径缺失**：子页面缺乏标准化的面包屑导航与返回按钮机制
6. **死链接与错链接**：文件迁移过程中产生大量断链与路径错误

为系统性解决上述问题，特编制本架构规划说明书，对V2.1版本的导航体系与引擎排布进行全局重构。

### 1.3 适用范围

本规范适用于Game-OS V2.1版本下所有页面，包括但不限于：

- 总控台首页（index.html）
- Y.Mine Studio创作台（ymine-studio.html）
- 六大子引擎页面（GameMind、AirMind、EvolveMind、MoodMind、MindSpeak、几何心智空间）
- 四大实验室板块（力学实验室、营销实验室、证据实验室、结构力学实验室）
- 各类实验详情页（#30重力、#31原子、#32分子等）
- 所有文档与说明页面

### 2.3 架构设计原则

1. **基座优先原则**：GameMind作为全系统公共博弈基座，在视觉与逻辑层级上突出显示
2. **同级平行原则**：四大业务子引擎（AirMind、EvolveMind、MoodMind、MindSpeak）为同级平行关系，采用统一卡片尺寸横排展示
3. **底层隐藏原则**：几何心智空间作为算力底座，不与业务子引擎并列展示，置于底层支撑位置
4. **唯一入口原则**：总控台为全站唯一入口，仅首页保留完整侧边栏导航
5. **路径可溯原则**：所有子页面必须提供面包屑导航与双重返回按钮
6. **公私分离原则**：公开演示层与SEALED涉密内核严格物理隔离

## 3. GameMind与四大子引擎架构规划方案

### 3.1 GameMind定位：全系统通用底层公共博弈基座

#### 3.1.1 核心定位

GameMind是Game-OS V2.1的全域通用公共博弈基座，定位为全系统所有子引擎与业务模块共享的底层算法内核。它不直接面向最终用户提供业务功能，而是为上层所有模块提供统一的博弈论、风控、决策算法支撑。

#### 3.1.2 核心算法模块

| 算法模块 | 核心功能 | 数学基础 | 服务对象 |
|---------|---------|---------|---------|
| 纳什均衡求解器 | 多主体博弈均衡点计算 | 不动点定理·纳什存在性定理 | 所有子引擎 |
| 红蓝博弈引擎 | 对抗场景推演与最优策略 | 零和博弈·极小极大定理 | AirMind·EvolveMind |
| 凯利公式计算器 | 最优仓位/资源配比计算 | 凯利判据·对数效用最大化 | 所有涉及资源分配模块 |
| 三圈演化模型 | 认知-策略-生态三层演化 | 演化博弈论·复制子动态 | EvolveMind·MoodMind |
| 圆锥风控内核 | 全局熔断与风险量化 | C值圆锥模型·正态分布1σ边界 | 全系统风控中间件 |

#### 3.1.3 关键代码配置

```javascript
// GameMind核心阈值配置 - models/gamemind/gamemind-config.js
const GAMEMIND_CONFIG = {
    CORE_THRESHOLDS: {
        SAFE: 0.48,           // 安全阈值
        WARNING: 0.50,        // 预警阈值（半凯利）
        DANGER: 0.68,         // 熔断阈值（1σ边界）
        ABSOLUTE_HARD_CAP: 0.25 // 绝对硬顶
    },
    KELLY_PARAMS: {
        FRACTIONAL_KELLY: 0.5, // 半凯利策略
        MAX_POSITION: 0.25,    // 单标的最大仓位
        DRAWDOWN_LIMIT: 0.40   // 最大回撤限制
    },
    SERVICES: {
        AIRMIND: true,
        EVOLVEMIND: true,
        MOODMIND: true,
        MINDSPEAK: true
    }
};
```

#### 3.1.4 服务调用关系

```
GameMind
  ├─→ 为AirMind提供：空域博弈定价·冲突解脱算法·调度最优解
  ├─→ 为EvolveMind提供：认知演化博弈·策略选择压力·适应度计算
  ├─→ 为MoodMind提供：情绪博弈定价·非标资产估值锚点·风险溢价计算
  └─→ 为MindSpeak提供：跨域映射权重·翻译置信度博弈·同构稳定性校验
```

### 3.3 几何心智空间算力底座定位（底层算力基座，非平行子引擎层）

#### 3.3.1 定位说明

几何心智空间（geom-compute）在架构中定位为**底层算力基座**，不属于四大平行子引擎层级。它不直接面向业务场景提供功能，而是为上层所有引擎提供：

1. 全域大模型算力错峰调度
2. 二进制底层执行环境
3. 高维几何计算加速
4. 算力资源动态分配
5. 异构计算统一抽象

#### 3.3.2 为何不与子引擎并列

| 维度 | 四大子引擎 | 几何心智空间 |
|------|-----------|-------------|
| 用户可见性 | 直接面向用户，有独立UI | 后台支撑，用户无感知 |
| 业务属性 | 解决具体业务问题 | 提供算力基础设施 |
| 交互方式 | 有独立页面与交互控件 | 无独立交互界面，API调用 |
| 展示层级 | 总控台卡片横排展示 | 底部小字标注，不做突出展示 |
| 依赖方向 | 依赖几何心智空间算力 | 被所有子引擎依赖 |

#### 3.3.3 视觉呈现规范

- 总控台页面中，几何心智空间不作为主要卡片展示
- 在页面底部技术支撑区域以小字标注："几何心智空间算力底座提供全域计算支持"
- 可保留独立访问页面（engines/geom-compute/index.html）供技术调试使用，但不在主导航中突出
- 不分配专属的动效与高亮配色

### 3.4 排布取舍逻辑说明：为什么GameMind突出展示为大卡片，四大子引擎统一卡片横排

#### 3.4.1 GameMind大卡片设计理由

1. **架构层级差异**：GameMind是基座，四大子引擎是上层应用，基座在视觉上需要更厚重以体现支撑感
2. **依赖关系体现**：所有子引擎都依赖GameMind的算法服务，居中大卡片体现"核心"地位
3. **功能广度差异**：GameMind服务全系统所有模块，四大子引擎各自只负责一个业务领域
4. **视觉重心需要**：总控台需要一个视觉锚点，避免五个相同大小卡片造成的视觉散漫
5. **用户认知引导**：新用户进入系统首先理解"这是一个博弈论为核心的系统"，再了解各业务引擎

#### 3.4.2 四大子引擎统一卡片横排理由

1. **架构同级性**：四大子引擎在调用关系上完全平行，无主次之分
2. **公平性体现**：统一尺寸体现各引擎同等重要，避免厚此薄彼
3. **扫描效率优化**：横排等宽卡片便于用户快速浏览对比
4. **扩展便利性**：未来新增第五、第六个子引擎时，横排布局可自然延伸或换行
5. **视觉节奏感**：一大四小的布局形成主次分明的视觉节奏，避免单调

#### 3.4.3 布局规格参数

| 元素 | 宽度占比 | 高度 | 位置 | 特殊样式 |
|------|---------|------|------|---------|
| GameMind卡片 | 100% | 280px | 顶部居中 | 更大标题·基座标识·呼吸光晕动效 |
| 四大子引擎卡片 | 每个24%（间距1%） | 220px | GameMind下方横排 | 统一尺寸·各自专属配色·悬停动效 |
| 几何心智空间 | 文字标注 | 行高 | 页脚区域 | 小号字体·灰色 |

### 4.1 总控台侧边栏规范（仅首页保留完整侧边导航）

#### 4.1.1 侧边栏保留范围

- 节点脉冲：当前激活节点周期性脉冲

| 色彩角色 | 色值 | 用途 |
|---------|------|------|

- 情绪波形：正弦波式起伏动画

| 色彩角色 | 色值 | 用途 |
|---------|------|------|

**专属动效：**
- 数据流动：源域→目标域的粒子流动
- 同构闪光：映射完成时绿色对勾闪光

### 5.3 动效性能规范

- 所有动画必须使用 CSS3 transform/opacity，避免触发重排
- 动画帧率目标：60fps
- 动效时长：基础动画200-300ms，呼吸类循环动画3-5s
- 提供 `prefers-reduced-motion` 媒体查询适配，尊重用户系统设置
- 页面滚动时自动暂停非关键动画

## 7. 验收标准

### 7.1 架构排布验收标准

| 验收项 | 验收标准 | 验证方法 |
|--------|---------|---------|

### 7.2 导航系统验收标准

| 验收项 | 验收标准 | 验证方法 |
|--------|---------|---------|
| 侧边栏范围 | 仅index.html和ymine-studio.html有侧边栏，其他页面无 | 逐页检查+资源引用检查 |
