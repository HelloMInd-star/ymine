## 1. 五层架构设计理念：公共基座→子引擎→实验室→公共资源→文档

### 1.1 设计哲学

Game-OS V2.1采用自下而上的五层分层架构设计，遵循"高内聚、低耦合、公私分离、依赖单向"的核心原则。架构设计参考了操作系统内核分层思想，将系统从底层资源到上层应用严格分离，确保：

1. **依赖单向性**：上层可以依赖下层，下层严禁反向依赖上层
2. **公私分离性**：公开演示层与SEALED涉密内核物理隔离
3. **职责单一性**：每个目录只负责一个明确的功能域
4. **可扩展性**：新增子引擎或实验室无需修改现有架构
5. **可维护性**：目录命名清晰，新人可快速理解代码组织

### 1.2 五层架构全景图

```
┌─────────────────────────────────────────────────────────────────┐
│                    L5 官方文档层 (docs/)                         │
│  审计报告 · 设计说明书 · 验收文档 · 架构规划 · 涉密资产清单        │
├─────────────────────────────────────────────────────────────────┤
│                    L4 公共资源层 (assets/, models/)              │
│  全局CSS · 公共JS · 数据模型 · 配置文件 · 中间件 · 工具函数        │
├─────────────────────────────────────────────────────────────────┤
│                    L3 实验室层 (labs/)                           │
│  力学实验 · 营销实验 · 证据实验 · 结构力学实验 · 各类沙箱演示      │
├─────────────────────────────────────────────────────────────────┤
│                    L2 子引擎层 (engines/)                        │
│  GameMind · AirMind · EvolveMind · MoodMind · MindSpeak ·        │
│  geom-compute（算力底座）· 各引擎SEALED私有内核                    │
├─────────────────────────────────────────────────────────────────┤
│                    L1 总控台层 (根目录)                           │
│  index.html · ymine-studio.html · 全局入口 · 旧版重定向页         │
└─────────────────────────────────────────────────────────────────┘
```

### 1.3 各层依赖规则

| 层级 | 可以依赖 | 不可以依赖 |
|------|---------|-----------|
| L1总控台层 | L2, L3, L4, L5 | 无（顶层） |
| L2子引擎层 | L4, L5 | L1, L3 |
| L3实验室层 | L2, L4, L5 | L1 |
| L4公共资源层 | L5 | L1, L2, L3 |
| L5官方文档层 | 无（纯文档） | L1, L2, L3, L4 |

### 1.4 架构演进说明

本次重构从V2.0的扁平结构演进为V2.1的五层结构，主要变更：

| 版本 | 架构特点 | 问题 |
|------|---------|------|
| V2.0 | 所有HTML文件散落在根目录 | 文件杂乱、职责不清、公私混放 |
| V2.1 | 五层严格分层 | 结构清晰、职责明确、公私分离 |

## 3. 每层详细说明（含README内容概述）

### 3.1 L1总控台层（index.html, ymine-studio.html）

#### 3.1.1 层级定位

L1层是系统的顶层入口层，位于仓库根目录，是用户访问系统的第一入口。该层仅保留两个核心HTML文件，严禁在根目录堆放业务页面。

#### 3.1.2 包含文件说明

| 文件 | 路径 | 功能说明 |
|------|------|---------|
| 总控台首页 | `/index.html` | 系统主入口，展示GameMind+四大子引擎卡片、实验室入口、熔断状态、数据流可视化。唯一保留完整侧边栏的页面。 |
| Y.Mine Studio | `/ymine-studio.html` | 创作工作台入口，同样保留侧边栏，提供创作类功能入口。 |

#### 3.1.3 L1层规范

- 根目录仅允许放置入口文件、顶级README、重定向页
- 业务页面必须下沉到L2或L3对应目录
- 重定向页仅做跳转用，不包含业务逻辑
- 根目录HTML文件可以引用L4层资源，可以链接到L2/L3层页面

#### 3.1.4 资源引用规范（index.html）

```html
<!-- L1页面可以引用L4层所有公共资源 -->
<link rel="stylesheet" href="../../assets/css/base.css">
<link rel="stylesheet" href="../../assets/css/poker-egg.css">
<script src="../../assets/js/bus.js"></script>
<script src="../../assets/js/sidebar.js"></script>
<script src="../../models/common/normalizer.js"></script>
<script src="../../models/gamemind/gamemind-config.js"></script>
<!-- 其他models... -->
```

### 3.2 L2子引擎层（engines/下每个引擎的公私目录划分）

#### 3.2.1 层级定位

L2层包含所有独立平行子引擎，是系统的核心业务逻辑层。每个子引擎一个独立目录，严格区分公开演示层与SEALED涉密内核。

#### 3.2.2 子引擎目录规范

每个标准子引擎目录结构：

```
engines/{engine-name}/
├── README.md           # 引擎说明文档
├── index.html          # 公开演示入口页面
├── assets/             # 引擎专属公开资源（可选）
│   ├── css/
│   └── js/
└── modules/            # 引擎专属公开模块（可选）
```

对应的SEALED涉密内核目录命名为 `{engine-name}-private-engine/`：

```
engines/{engine-name}-private-engine/
├── README.md           # 涉密说明（标注SEALED_FOR_COPYRIGHT_DEPOSIT）
├── index.js            # 内核入口
└── *.js                # 涉密算法实现文件
```

#### 3.2.3 现有子引擎清单

| 目录 | 引擎名称 | 版本 | 公开入口 | 涉密内核 | 状态 |
|------|---------|------|---------|---------|------|
| gamemind/ | GameMind博弈基座 | V1.0 | index.html | 待建设 | 已部署 |
| airmind/ | AirMind低空调度 | V2.0 | index.html | airmind-private-engine/ | 已部署 |
| evolvemind/ | EvolveMind认知演化 | V1.0 | index.html | 待建设 | 已部署 |
| moodmind/ | MoodMind非标估值 | V1.0 | index.html | 待建设 | 已部署 |
| mindspeak/ | MindSpeak跨域翻译 | V19.0 | index.html | mindspeak-private-engine/ | 已部署 |
| geom-compute/ | 几何心智空间算力底座 | V1.0 | index.html | 不需要（纯算力） | 已部署 |

#### 3.2.4 资源引用规范（子引擎页面）

子引擎页面引用公共资源时必须使用**相对路径回到根目录**：

```html
<!-- engines/gamemind/index.html 中引用L4资源 -->
<link rel="stylesheet" href="../../assets/css/base.css?v=2">
<script src="../../assets/js/bus.js?v=3"></script>
<script src="../../models/gamemind/gamemind-config.js?v=3"></script>

<!-- 子引擎页面严禁引入sidebar.js -->
<!-- <script src="../../assets/js/sidebar.js"></script>  禁止！ -->
```

#### 3.2.5 子引擎页面必备元素

每个子引擎index.html必须包含：

1. 面包屑导航：`总控台 > 子引擎中心 > [引擎全称+版本号]`
2. 双返回按钮：返回上一级 + 返回总控台
3. 引擎专属配色与样式
4. 引擎功能演示区域
5. 数据状态展示（如适用）

### 3.3 L3实验室层（labs/下各类实验分类）

#### 3.3.1 层级定位

L3层包含各类实验、沙箱、演示、证据展示页面，是验证算法、展示效果的实验场所。实验室页面可以调用L2子引擎和L4公共资源，但L2子引擎严禁依赖L3实验室。

#### 3.3.2 实验室分类

| 实验室目录 | 名称 | 包含内容 | 入口文件 |
|-----------|------|---------|---------|
| structural-mechanics/ | 结构力学实验室 | #30重力、#31原子、#32分子实验 | exp30-gravity.html |
| marketing/ | 营销实验室 | 营销重构、定价模型实验 | index.html |
| evidence/ | 证据实验室 | 各类Benchmark、案例库、证据展示 | lab-section.html |

#### 3.3.3 实验页面命名规范

- 编号实验：`exp{编号}-{名称}.html`，如 `exp30-gravity.html`
- 通用实验：`{功能描述}.html`，如 `ai-pricing-benchmark.html`
- 实验室入口：`index.html` 或 `lab-section.html`

#### 3.3.4 实验室分类README内容要点

每个实验室子目录README.md应包含：

1. 实验室定位与研究方向
2. 包含实验列表及简介
3. 实验数据输出key说明
4. 依赖的子引擎列表
5. 运行说明

### 3.4 L4公共资源层（assets/, models/）

#### 3.4.1 层级定位

## 4. 公私目录隔离规范（公开层vs SEALED私有内核）

### 4.1 隔离原则

所有涉密算法、核心权重、私有推导过程必须放在 `*-private-engine/` 目录下，与公开演示层严格物理隔离。公开层代码不得直接引用私有内核文件。

### 4.2 目录命名约定

| 类型 | 目录命名 | 可见性 | 版权标记 |
|------|---------|--------|---------|
| 公开演示层 | `engines/{name}/` | 公开可访问 | 无 |
| 私有内核 | `engines/{name}-private-engine/` | 涉密，版权保护 | SEALED_FOR_COPYRIGHT_DEPOSIT |

### 4.3 私有内核目录规范

每个private-engine目录必须：

1. 在README.md顶部标注：
```markdown
# SEALED_FOR_COPYRIGHT_DEPOSIT

本目录包含版权保护的涉密算法实现，未经授权不得复制、分发、反编译。
```

2. 通过 `private-engine-loader.js` 或统一的加载器机制按需加载
3. 涉密文件不得被公开层HTML直接通过`<script>`标签引用
4. 核心算法代码可经过混淆或加密处理

### 4.4 加载机制规范

正确的公私分离加载模式：

```
公开层 (index.html)
    ↓ 调用
公开适配层 (mindspeak-public.js)
    ↓ 鉴权/加载
私有内核加载器 (private-engine-loader.js)
    ↓ 安全加载
SEALED涉密内核 (*-private-engine/*.js)
    ↓ 返回
计算结果（公开数据结构）
```

错误做法（禁止）：

```html
<!-- 禁止：公开HTML直接引用涉密内核 -->
<script src="../../engines/mindspeak-private-engine/confidence-kernel.js"></script>
```
