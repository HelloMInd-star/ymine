## 一、项目核心思想与底层逻辑

### 1.1 六层金字塔架构

Y.Mine 不是知识堆砌，而是一套**演绎法驱动的精算操作系统**——从第一性原理出发，通过神经科学×金融精算的交叉映射，把"定价权"从经验直觉转化为可计算变量：

```
🧮 L5 精算输出层       CAPM 资本定价 → 凯利仓位 → 风控熔断
       ↑ Rp × β
⚙️  L4 核心算法层       6维风险向量 + 圆锥浓度模型 + 三流分发
       ↑ f*, L, SR, mdd, dim, oc
🔗  L3 证据链层         跨领域吞噬 + 商业证据闭环 + 价值金字塔
       ↑ 元架构迁移
🧠  L2 思维底座层       GameMind(博弈) + MoodMind(估值) + EvolveMind(认知)
       ↑ 神经科学映射
🧪  L1 信号采集层       7层信息漏斗 + 画圈实验行为代理
       ↑ 100%噪声→20%信号
🛠️  L0 工具验证层       二元拆分 + 语言映射 + 低空经济落地验证
```

### 1.2 核心数学公式集

所有业务逻辑严格锚定以下数学结构，不引入任何主观加权：

| 模型 | LaTeX 公式 | 物理意义 |
|------|-----------|----------|
| **CAPM 资本定价** | $$R = R_f + R_p \cdot \beta$$ | 无风险利率 + 风险溢价 × 系统性风险 |
| **圆锥浓度模型** | $$C = \frac{m \cdot \rho_0}{V(h)}$$ | 溶质质量×信源密度/市场容积，对应市场份额 |
| **6维风险向量模长** | $$\|\mathbf{V}\| = \sqrt{f^{*2} + L^2 + SR^2 + mdd^2 + dim^2 + oc^2}$$ | 凯利/杠杆/夏普/回撤/递减/机会成本的欧氏模长 |
| **余弦方向相似度** | $$\cos\theta = \frac{\mathbf{V} \cdot \mathbf{V}_{opt}}{\|\mathbf{V}\| \cdot \|\mathbf{V}_{opt}\|}$$ | 与理论最优向量的方向对齐度 |
| **凯利最优仓位** | $$f^* = \frac{b \cdot p - q}{b}$$ | 赔率b、胜率p下的对数最优增长率 |
| **经营杠杆 DOL** | $$DOL = \frac{CM}{\pi} = \frac{P - VC}{P - VC - FC}$$ | 边际贡献/营业利润，度量固定成本放大效应 |
| **营销ROI (ROMI)** | $$ROMI = \frac{CM}{TAC}$$ | 边际贡献/总获客成本，度量营销投入效率 |
| **综合真实ROI** | $$ROI_{real} = \frac{ROMI \cdot L}{1 + DOL \cdot 0.1}$$ | 营销回报经财务杠杆放大并经营杠杆折减 |

### 1.3 三条稳态铁律

浓度 $C$ 的三条阈值线是整个系统的风控基石（来自金融市场正态分布的 $1\sigma$ 稳态区间）：

| 阈值 | 含义 | 操作指令 |
|------|------|----------|
| $C < 0.48$ | 生死线以下，无定价权 | 🔵 观望试探，仅20%凯利仓位 |
| $0.48 \leq C \leq 0.50$ | 保本中轴，均衡稳态 | 🟢 主力重仓，完整凯利仓位 |
| $0.50 < C \leq 0.68$ | 边际效用递减区 | 🟡 缩减50%投放规模 |
| $C > 0.68$ | 过热熔断线 | 🔴 二进制熔断，禁止新增资源 |

## 二、深度审计与量化评估

### 2.1 审计时间点：v1.3 容器扩充版

v1.3 阶段项目已积累15个HTML页面、16个案例、完整的6维向量+圆锥+CAPM+凯利公式业务逻辑，但**工程结构处于原始状态**：

### 2.2 P0级致命缺陷（数据流断层）

| 缺陷 | 量化影响 | 根因 |
|------|---------|------|
| **营销-金融割裂** | ROMI、DOL、杠杆ROI完全不可计算 | 定价模型的"基础成本"是孤立滑块，未包含营销成本；营销理论（三流分发）仅为文字对照，无财务化映射 |
| **侧边栏重复15次** | 每个HTML内联~120行侧边栏HTML，共1800行冗余；新增菜单需改15个文件 | 无公共资源抽离，复制粘贴式开发 |
| **光晕效果重复15次** | mousemove 事件监听器在每个页面独立绑定，~40行/页，共600行冗余 | 同上 |
| **全局CSS重复15次** | reset/body/sidebar/glow/animation ~80行/页，共1200行冗余 | 无公共CSS |

### 2.3 P1级严重问题（初始化不一致）

| 缺陷 | 量化影响 |
|------|---------|
| **localStorage 无统一初始化** | 9个通道各自读写，首次访问时部分通道为 `null`，导致JSON.parse报错 |
| **跨页数据同步手写** | 每个引擎各自监听 `storage` 事件，命名不一致（`funnelOutput` vs `capmAutoMode`） |
| **内联IIFE闭包冲突风险** | 所有业务逻辑包在IIFE中，但缺乏统一命名空间保护 |

### 2.4 数据流健康度评分（v1.3）

| 链路 | 健康度 | 状态 |
|------|--------|------|
| 漏斗→CAPM覆盖 | 80% | ✅ 已有覆盖逻辑 |

| 圆锥浓度→CAPM | 90% | ✅ concentrationFactor已实现 |

## 四、v1.6 核心突破：业财融合与数据串联

> **目标：打通审计报告中标红的三条0%链路——营销投入→真实成本、营销效率→机会成本、营销ROI→风险溢价，将数据流健康度从46%提升至95%+。**

### 4.1 破局思路：三流分发的财务化映射

原系统中"三流分发"（信息分发/流量分发/物理分发）只是营销学概念的静态表格对照。v1.6 将其严格映射为**可计算的财务成本结构**，核心创新在于引入**机会成本的三维度递减函数**：

```
营销三维度 → 财务成本映射
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
维度1 · 品牌认知 C₁（信息分发/触达率r₁）
   直接成本：内容制作 + 渠道投放
   机会成本：OC₁ = C₁ · r₁ · Rf     ← 资金时间价值（无风险利率）

维度2 · 市场转化 C₂（流量分发/转化率r₂）
   直接成本：促销补贴 + KOL佣金 + 运营人力
   机会成本：OC₂ = C₂ · r₂ · DIM    ← 边际递减率（规模扩张衰减）

维度3 · 履约交付 C₃（物理分发/履约率r₃）
   直接成本：物流 + 售后 + 客服体系
   机会成本：OC₃ = C₃ · r₃ · MDD    ← 最大回撤容忍度（履约失败损失）
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

### 4.2 业财融合核心公式链

以总获客成本 $TAC$ 为枢纽，串联起营销与金融两个世界：

$$
\begin{align}
MC_{total} &= C_1 + C_2 + C_3 \tag{直接营销成本} \\
OC_{total} &= C_1 r_1 R_f + C_2 r_2 \cdot DIM + C_3 r_3 \cdot MDD \tag{机会成本} \\
TAC_{unit} &= MC_{total} + OC_{total} \tag{单件获客成本} \\
N_{exp} &= 1000 \cdot r_1 \cdot r_2 \cdot r_3 \tag{预期成交单量} \\
C_{real} &= C_{base} + TAC_{unit} \tag{真实单位成本} \\
P &= C_{real} \cdot premium \cdot (1 + elasticity \cdot 0.2) \tag{基于真实成本的定价} \\
CM &= P - C_{real} \tag{边际贡献} \\
ROMI &= \frac{CM}{TAC_{unit}} \tag{营销投资回报率} \\
DOL &= \frac{P}{CM} = \frac{1}{grossMargin} \tag{经营杠杆} \\
ROI_{real} &= \frac{ROMI \cdot L}{1 + DOL \cdot 0.1} \tag{杠杆后真实ROI}
\end{align}
$$

### 4.3 数据流闭环（v1.6 新增第10条总线通道）

```javascript
// bus.js 新增 marketingFinance 通道
marketingFinance: {
  storageKey: 'marketingFinance',
  eventName: 'marketing-finance-update',
  defaultValue: {
    C1, C2, C3, r1, r2, r3,       // 输入：营销三维度投入+效率
    MC_total, OC_total, TAC,       // 中间：成本拆解
    realUnitCost, N_expected,      // 中间：真实成本+预期单量
    ROMI, DOL, totalLeverage,      // 输出：财务指标
    realROI, status, timestamp     // 输出：综合ROI+风控状态

```

**三向串联逻辑（不改原有公式，仅外层包裹）**：

```
                    ┌──────────────────────────┐
                    │  📊 业财融合面板（新增）    │
                    │  C1,C2,C3,r1,r2,r3 → TAC  │

                           │ YBus.publish('marketingFinance')
                           ▼
        ┌──────────────────┼──────────────────┐
        │                  │                  │
        ▼                  ▼                  ▼
┌──────────────┐  ┌──────────────┐  ┌──────────────┐
│ 💰定价模型    │  │ 📊6维风险向量  │  │ 📐CAPM引擎    │
│ cost→realCost│  │ oc滑块自动同步│  │ Rp ×romiFactor│

└──────┬───────┘  └──────┬───────┘  └──────┬───────┘
       │                 │                  │
       └─────────────────┼──────────────────┘

                  ┌──────────────┐
                  │ ⚠️凯利资源配置 │
                  │  f*自动更新   │
                  └──────────────┘
```

### 4.4 串联实现的关键设计决策

| 决策 | 选择 | 理由 |
|------|------|------|
| **注入方式** | 不修改滑块值，在 `updatePricing()` 内优先读 `window._marketingFinanceData` | 避免循环触发；保持用户手动调成本的能力 |
| **OC同步** | `normalizedOC = OC_total / MC_total × 0.5`，裁剪到[0.05, 0.70] | 把绝对机会成本归一化为0~1的风险向量维度 |
| **ROMI→Rp修正** | `romiFactor = 1 + (1 - ROMI_clamped) × 0.15`，裁剪到[0.8, 1.2] | ROMI高说明营销有效→降低风险溢价；亏损状态→提高风险溢价 |
| **风控阈值** | 亏损/跑输Rf/健康(Rf~100%)/过热(>300%杠杆)/高效 五档 | 不挪用0.68浓度铁律，为ROI单设300%过热线（约3倍杠杆年化） |

**目标：把金融估值模拟器中的6维向量/圆锥/CAPM/凯利四个核心引擎，从内联IIFE抽离为独立的 `assets/js/actuarial-engine.js`**

### v1.8 全链路从漏斗开始打通（计划中）

**目标：让信息漏斗的7层筛选结果真实驱动下游每一个参数，而不是仅覆盖Rf/Rp/β**

- 漏斗L2信源密度 $\rho_0$ → 自动同步到圆锥浓度面板
- 漏斗L4黑天鹅信号 → 自动推高MDD（最大回撤）滑块
- 漏斗L7最终剪辑率 → 自动修正综合成功率
- 业财融合ROMI信号 → 反向回流到漏斗L1的"资源分配建议"
- 实现真正的**双向闭环数据流**

### v2.0 愿景：可嵌入的精算SDK

- 将整个精算系统封装为 `<ymine-actuarial>` Web Component
- 对外暴露统一API：`YMine.price(params)` / `YMine.risk(vector)` / `YMine.kelly(odds, prob)`
- 支持嵌入任意第三方页面
- 保持零依赖：单个JS文件 < 50KB
