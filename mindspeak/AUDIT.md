## 三、公私资产划分审计

### 3.1 验收标准
- [x] 目录严格二分：`mindspeak/`（公开）和 `mindspeak-private-engine/`（私有）
- [x] 公开代码可公示，不包含涉密精算逻辑
- [x] 私有内核仅暴露密封接口桩，前端不访问内部实现
- [x] localStorage命名空间统一前缀`mindspeak_*`
- [x] 软著申报材料清晰区分公开/涉密清单

### 3.2 公私资产划分明细

#### ✅ 公开目录（mindspeak/）审计
- 主页面index.html：纯UI框架与交互逻辑，无涉密权重数值
- ms-samples.js：50组对照样本数据，全部为教学演示用例
- ms-visualization.js：Canvas可视化算法，纯前端渲染逻辑
- mindspeak-public.js：公开层核心逻辑，仅调用私有内核接口，不包含精算实现

**审计结论：** 公开目录无涉密内容泄露，可安全用于GitHub公开展示和软著正常登记。

#### ✅ 私有涉密目录（mindspeak-private-engine/）审计
| 文件 | 涉密标记 | 接口暴露检查 | 源码泄露检查 |
|------|---------|-------------|-------------|
| vector4d-operator.js | 绝密 | 仅暴露createVector/transform等桩方法 | ✅ 完整微分算子已标记SEALED |
| weight-matrix.js | 绝密 | 仅暴露getPublicWeights公开权值 | ✅ 完整10×10×4张量已密封 |
| confidence-kernel.js | 绝密 | 仅暴露evaluate/decay接口 | ✅ 贝叶斯/卡尔曼内核已密封 |
| crypto-matrix.js | 机密 | 仅暴露sign/encrypt桩方法 | ✅ RSA/AES完整实现已密封 |

**审计结论：** 私有内核所有文件均为接口桩形式，完整精算逻辑已标记"SEALED_FOR_COPYRIGHT_DEPOSIT"，前端无法通过任何方式获取涉密源码，满足软著例外交存要求。

#### ✅ 存储命名空间审计
- 所有localStorage键名均以`mindspeak_`为前缀
- pipeline/draft/audit三分区权限隔离正确
- audit_log_mindspeak仅追加不可篡改（前端无删除接口）
