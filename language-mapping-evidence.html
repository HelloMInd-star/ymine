🧭 这是什么
一个“结构即算法”的交互式验证台。
把语言结构（语序/时态/词缀/被动/多义词/文化/边界）
映射到数学结构（线性代数/凯利公式/二元拆分/PACD）
再映射到算法语言（交换律/概率/特征工程/IoC/策略/正则化/异常检测）。
🧠 核心思想
所有符号系统（语言、数学、代码）
共享同一套底层逻辑：
生成候选 → 验证约束 → 冲突回退 → 采纳最优
这个页面就是用来验证这个假设的。

📊 跨域映射全景
④ 词缀派生 → 特征工程
makan → makanan / pemakan / memakan
基特征 → 多项式特征 / 归一化 / 交互特征
💡 词根 + 词缀 = 新词，类比基础特征 + 特征变换 = 新特征
⑤ 被动句式 → 控制反转 (IoC)
Saya makan nasi → Nasi dimakan oleh saya
调用者 → 被调用者 (依赖注入 / DI)
💡 主动→被动 = 控制反转 (IoC)，数据流方向反转
⑥ 多义词消歧 → 策略模式
"打" → hit / play / take (上下文决定)
不同上下文 → 不同算法 (Strategy Pattern)
💡 根据上下文选择词义 = 根据上下文选择算法
⑦ 文化适配 → 正则化
"龙" → "Loong" (避免西方负面联想)
L1/L2 正则化 → 约束模型输出
💡 文化约束 = 正则化约束，防止不当输出
⑧ 差异边界 → 异常检测
"kereta" → 马来西亚=汽车 / 印尼=火车
OOD 检测 → 置信度降级 / 人工介入
💡 伪同源词 = 分布外 (OOD) 数据 → 触发异常处理
💡 每一层语言证据都对应一类算法设计思想，构成从“语言学”到“计算机科学”的完整映射链。
